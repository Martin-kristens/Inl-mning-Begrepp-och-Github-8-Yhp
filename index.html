Begrepp och GitHub

GitHub Flow

Visar ett grenbaserat arbetsflöde för programmerare som arbetar i Github. Github är en molnbaserad webbsida vart man kan lagra sina projekt. Ett program som låter flera programmerare ta del av och ändra i projekt. Man använder det genom att man skapar ett repository i github, klonar ner det till sin lokala dator. När man arbetar i sin lokala dator och pushar man upp sina filer i github, detta kan flera programmerare göra samtidigt. Man kan sedan granska varandras arbeten, ser man att det inte finns någon konflikt kan man merga (föra samman) samtliga filer till en master. Dessa kan man sedan publisera med hjälp av github pages.

Flexbox

Hjälper programmeraren att strukturera innehållet och bygga layouten på webbsidan samt anpassar webbsidan så att den passar olika skärmstorlekar. Man arbetar mycket med att skapa boxar som positioneras, både rader och kolumner. Man skapar flexbehållare (display: ) som styr över underordnade flexelement. Man kan även styr de olika elementens positionering.

DOM

Document Object Model HTML trädet där alla element är samlade och visas i webbläsaren. Man hittar trädet genom att högerklicka på webbsidan, klickar på inspektera knappen så kommer trädet upp. Visar strukturen och hur man kan manipulera det. Genom att tilldela element ID samt klasser och sätta namn på dessa kan man se dessa i trädet. ID är specefika medans klasser kan innefatta flera element. Dessa ID och klasser kan man sedan kalla på i JavaScript genom att skriva "document.getElementById();" alt "document.getElementByClass();". Därefter kan man ändra eller lägga till kod.

Fin kod

Koden man skriver ska vara tydlig, strukturerad och lättläst. Varför detta är viktigt är för att andra ska kunna snabbt sätta sig in i projektet samt om det dröjer lång tid innan man själv tittar på koden efter man skrivit den. Även viktigt med kommentarer för både sig själv och andra.

Ramverk

Färdiga kod-stycken man kan hämta hem till sin egen för att boosta layouten. Ex Bootstrap, detta ramverk arbetar främst mot mobilanvändandet och front-end utveckling. Flexbox kallas även bibliotek, detta ramverk arbetar också för en responsiv webbsida, skillnaden mot Bootstrap är att flexbox är integrerat i flertalet webbläsare därav att man inte behöver länka till någon speciell sida eller kod.
